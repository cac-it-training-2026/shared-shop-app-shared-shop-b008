# [Feature] お届け先登録機能の実装

## 概要 (Overview)

ユーザーの利便性向上を目的として、会員ごとに複数のお届け先を登録できる機能を実装します。

会員詳細画面からお届け先を最大3件まで登録・編集・削除できるようにしてください。

注文時のお届け先入力画面では、登録済みのお届け先を選択できるようにし、初期値は「お届け先1」とします。

また、お届け先選択欄の下に「新しい住所に届ける」リンクを配置し、登録済みのお届け先以外の住所を都度入力して注文できるようにしてください。

---

## 技術的制約・前提条件 (Constraints & Context)

- Spring Securityなどpom.xmlの変更を伴うライブラリを使用しないこと。
- サービスレイヤはなく、基本的なMVCで実装する（ControllerからJPAのメソッドを呼んでいます。）
- DBアクセスは Spring Data JPA を使用している前提でコードを生成すること。
- テンプレートエンジンは Thymeleaf を使用すること。
- お届け先は会員ごとに管理すること。
- 1会員につき登録できるお届け先は最大3件とすること。
- 注文時に登録済みお届け先または新規入力住所を選択できること。
- 既存の注文機能へ影響を与えないこと。
- ブランチ名は feat#nn のようにfeatの後にIssue番号をつけること。

---

## 受け入れ条件 (Acceptance Criteria)

- お届け先管理用テーブルが新規作成されていること。
- 会員詳細画面にお届け先管理エリアが追加されていること。
- 会員はお届け先を最大3件まで登録できること。
- 会員は登録済みお届け先を編集できること。
- 会員は登録済みお届け先を削除できること。
- 4件目を登録しようとした場合はエラーメッセージが表示されること。
- お届け先には氏名、郵便番号、住所、電話番号が保存されること。
- 注文時のお届け先入力画面で登録済みお届け先を選択できること。
- お届け先選択欄の初期値は「お届け先1」であること。
- 「新しい住所に届ける」リンクが表示されること。
- 「新しい住所に届ける」押下時、新規住所入力フォームへ遷移できること。
- 新規入力住所を使用して注文できること。
- 登録済みお届け先を選択して注文できること。
- 他ユーザーのお届け先情報が表示・編集できないこと。
- サーバ再起動後もお届け先情報が保持されること。

## 実装ステップ案 (Implementation Steps for Copilot)

AIアシスタントは以下のステップに従って実装と提案を行ってください。

### 1. DeliveryAddress Entity作成

お届け先を管理するEntityクラスを新規作成してください。

```java
@Entity
@Table(name = "delivery_address")
public class DeliveryAddress {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Integer id;

    private Integer userId;

    private Integer addressNo;

    private String name;

    private String postalCode;

    private String address;

    private String phoneNumber;
}
```

---

### 2. DBテーブル作成

お届け先管理用テーブルを作成してください。

```sql
CREATE TABLE delivery_address (
    id NUMBER PRIMARY KEY,
    user_id NUMBER NOT NULL,
    address_no NUMBER NOT NULL,
    name VARCHAR2(50) NOT NULL,
    postal_code VARCHAR2(10) NOT NULL,
    address VARCHAR2(255) NOT NULL,
    phone_number VARCHAR2(20) NOT NULL
);
```

必要に応じて users テーブルとの外部キー制約を設定してください。

### 4. 会員詳細画面改修

会員詳細画面に「お届け先管理」エリアを追加してください。

表示項目

- お届け先番号
- 氏名
- 郵便番号
- 住所
- 電話番号
- 編集ボタン
- 削除ボタン

また、お届け先登録ボタンを配置してください。

登録件数が3件の場合は登録ボタンを非活性化するか、エラーメッセージを表示してください。

---

### 5. お届け先登録画面作成

以下の入力フォームを作成してください。

- 氏名
- 郵便番号
- 住所
- 電話番号

登録時に現在のお届け先件数を確認し、3件を超える場合は登録できないようにしてください。

---

### 6. お届け先編集機能作成

登録済みお届け先の編集機能を実装してください。

編集対象のお届け先がログインユーザーの所有データであることを確認してください。

---

### 7. お届け先削除機能作成

登録済みお届け先の削除機能を実装してください。

削除後はお届け先一覧へ戻るようにしてください。

---

### 8. 注文画面改修

注文のお届け先入力画面に登録済み住所選択機能を追加してください。

表示例

```text
◉ お届け先1
○ お届け先2
○ お届け先3
```

初期選択は「お届け先1」としてください。

登録済み住所は氏名・住所が判別できる形式で表示してください。

---

### 9. 新しい住所に届ける機能

お届け先選択欄の下に以下のリンクを配置してください。

```text
新しい住所に届ける
```

押下時は新規住所入力画面へ遷移してください。

入力項目

- 氏名
- 郵便番号
- 住所
- 電話番号

この住所は注文時のみ利用し、お届け先マスタへ自動登録しないものとします。

---

### 10. 注文処理改修

注文確定時に以下を判定してください。

#### 登録済みお届け先選択時

- 選択されたお届け先情報を注文情報へ設定する。

#### 新規住所入力時

- 入力された住所情報を注文情報へ設定する。

既存の注文登録処理との整合性を保つこと。

---

### 11. バリデーション

お届け先登録・編集時に以下を実施してください。

- 氏名：必須
- 郵便番号：必須
- 住所：必須
- 電話番号：必須

既存会員登録時と同等の入力チェックを適用してください。
