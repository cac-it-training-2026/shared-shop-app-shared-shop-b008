# [Feature] 商品レビュー機能追加（評価集計機能含む）

| 項目 | 内容 |
|---|---|
| 概要 (Overview) | 購入者が購入した商品に対してレビューを投稿できる機能を追加する。注文詳細画面からレビュー入力画面へ遷移し、商品詳細画面およびトップ画面でレビュー情報と評価集計を確認できるようにする。 |
| 目的 | 購入者の評価を商品情報へ反映し、他ユーザーの商品購入判断材料として利用できるようにする。 |
| 技術構成 | Spring Boot / Java / Thymeleaf / Spring Data JPA / Oracle Database |
| 制約 | Service層は作成しない。ControllerからRepositoryを呼び出す既存MVC構成を維持する。Spring Security導入、pom.xml変更は禁止。 |
| 詳細仕様 | 注文詳細画面の商品名横に「レビューを書く」ボタンを追加する。 |
| 投稿条件 | 対象商品を購入済みユーザーのみレビュー投稿可能。ORDER_ITEMSを利用して購入確認を行う。 |
| レビュー項目 | 評価（1〜5必須）、レビュー本文（REVIEW_COMMENT・null可）、投稿日（INSERT_DATE自動登録）。 |
| 編集 | レビュー編集機能は実装しない。 |
| 削除 | 投稿者本人またはADMIN権限ユーザーのみ削除可能。 |
| DB利用 | 既存REVIEWSテーブルを利用する。 |
| REVIEWS項目 | ID、USER_ID、ITEM_ID、ORDER_ITEM_ID、RATING、REVIEW_COMMENT、INSERT_DATE。 |
| Entity | Review Entityを作成し、User・Item・OrderItemと関連付ける。 |
| Repository | 商品別レビュー取得、平均評価取得、件数取得、並び替え取得を実装する。 |
| 商品詳細画面 | 下部にレビュー一覧を表示する。投稿者名、星評価、コメント、投稿日を表示する。 |
| 並び替え | 新しい順を初期表示。評価が高い順、低い順へ変更可能。 |
| 表示件数 | 最新5件表示。「全件表示」ボタンで全レビュー表示。 |
| 評価集計 | 商品ごとの平均評価とレビュー件数を計算する。 |
| トップ画面 | 商品一覧に平均評価（★表示）とレビュー件数を表示する。 |
| 受け入れ条件 | 注文詳細からレビュー入力へ遷移できること。 |
| 受け入れ条件 | 未購入の商品にはレビュー投稿できないこと。 |
| 受け入れ条件 | 評価1〜5以外は登録できないこと。 |
| 受け入れ条件 | 商品詳細画面でレビュー確認できること。 |
| 受け入れ条件 | トップ画面で平均評価が表示されること。 |
| 実装ステップ | ①既存Entity確認 ②Review Entity作成 ③Repository作成 ④Controller追加 ⑤Thymeleaf修正 ⑥評価集計追加 ⑦削除権限追加 |
| ブランチ名 | feat#<Issue番号>形式で作成する。 |
