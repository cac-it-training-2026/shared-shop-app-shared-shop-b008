# [Feature] 最近見た商品の表示機能

## 概要 (Overview)

ユーザーが閲覧した商品の履歴を保存し、商品詳細画面のサイドバーに「最近見た商品」として表示する機能を実装する。

ユーザーが過去に閲覧した商品へ再度アクセスしやすくすることで、利便性の向上および購入促進を図る。

表示件数は最新5件までとする。

---

## 技術的制約・前提条件 (Constraints & Context)

- Spring Securityなどpom.xmlの変更を伴うライブラリは使用しないこと
- サービスレイヤは使用せず、既存のMVC構成（ControllerからRepositoryを呼び出す構成）に合わせて実装すること
- DBアクセスはSpring Data JPAを使用すること
- 閲覧履歴はデータベースで管理すること
- 閲覧履歴管理用のEntityを新規作成すること
- Entity追加に伴い、テーブル作成SQLおよびシーケンス作成SQLを作成すること
- 商品詳細画面表示時に閲覧履歴を登録すること
- 閲覧履歴はログインユーザーごとに管理すること
- 最近見た商品は既存の共通サイドバーに表示すること
- 表示件数は最新5件までとすること
- Spring Securityなど新規認証ライブラリは使用しないこと
- サービスクラスは作成しないこと
- 既存プロジェクトのコーディング規約およびGoogle Java Styleに準拠すること
- ブランチ名は `feat#nn` のように feat の後に Issue 番号をつけること

---

## 受け入れ条件 (Acceptance Criteria)

- 商品詳細画面を表示した際に閲覧履歴が登録されること
- 閲覧履歴がユーザーごとに管理されること
- 最新の閲覧商品から順番に表示されること
- 最近見た商品が最大5件表示されること
- 他ユーザーの閲覧履歴が表示されないこと
- ログアウト後も閲覧履歴が保持されること
- 閲覧履歴が存在しない場合でも画面エラーにならないこと
- 商品詳細画面のサイドバーに最近見た商品が表示されること
- 既存の商品一覧、商品詳細、カート、注文機能へ影響を与えないこと

## 実装ステップ案 (Implementation Steps for Copilot)

AIアシスタントは以下のステップに従って実装と提案を行ってください。

### 1. SQL作成

閲覧履歴管理用テーブルの作成SQLを作成してください。

また、以下も併せて作成してください。

- テーブル作成SQL
- シーケンス作成SQL
- 外部キー制約追加SQL
- テストデータ投入SQL（必要に応じて）
- DROP文

既存システムの命名規則に合わせること。

### 2. Entityの作成

閲覧履歴を管理するEntityクラスを新規作成してください。

最低限以下の情報を保持できること。

- ID
- 会員ID
- 商品ID
- 閲覧日時

既存のusersテーブルおよびitemsテーブルとの関連を考慮して設計してください。

### 3. Repositoryの作成

閲覧履歴の登録および取得を行うRepositoryを作成してください。

以下の処理を実装してください。

- 閲覧履歴登録
- ユーザーごとの履歴取得
- 最新5件取得

### 4. 商品詳細画面表示時の履歴登録処理実装

商品詳細画面を表示するControllerを改修してください。

商品詳細画面表示時に以下を実施してください。

- ログインユーザー情報を取得
- 閲覧履歴を保存

### 5. 最近見た商品の取得処理実装

ログインユーザーの閲覧履歴を取得し、閲覧日時の降順で並び替えて最新5件を取得してください。

### 6. サイドバー表示機能実装

既存の共通サイドバーを改修してください。

取得した閲覧履歴を利用し、以下を表示してください。

- 商品名
- 商品詳細画面へのリンク

画面変更を伴うため、変更箇所を視覚的に示してください。

### 7. 動作確認

以下の観点でテストを実施してください。

- 初回閲覧時の履歴登録
- 複数商品の閲覧
- 最新5件表示
- ユーザーごとの履歴管理
- ログアウト後の履歴保持
- 閲覧履歴未登録時の表示確認
- 既存機能への影響確認
