# [Feature] 商品レビュー機能追加（評価集計・購入者限定レビュー）

| 項目 | 内容 |
|---|---|
| 概要 (Overview) | 購入済みユーザーが商品に対してレビューを投稿できる機能を追加する。注文詳細画面からレビュー入力へ遷移し、商品詳細画面とトップ画面でレビュー情報を表示できるようにする。 |
| 目的 | 購入者の評価を商品情報へ反映し、他ユーザーの商品購入判断材料として利用できるようにする。 |
| 対象システム | Spring Boot / Java / Thymeleaf / Spring Data JPA / Oracle Database |
| 既存構成制約 | Service層は存在しないため新規作成禁止。ControllerからRepositoryを呼び出す既存MVC構成を維持する。 |
| DB制約 | 既存DB設計を変更しない。REVIEWSテーブルのCREATE TABLEは禁止。既存REVIEWSテーブル定義を確認してEntityを作成すること。 |
| 認証制約 | Spring Security導入禁止。pom.xml変更禁止。既存ログイン・Session管理方式を利用すること。 |
| ブランチ名 | feat#<Issue番号> 形式で作成する。 |


| 項目 | 内容 |
|---|---|
| DB利用 | 既存REVIEWSテーブルを利用する。 |
| REVIEWS構成 | ID / USER_ID / ITEM_ID / ORDER_ITEM_ID / RATING / REVIEW_COMMENT / INSERT_DATE を利用する。 |
| リレーション | USER_ID → USERS.ID、ITEM_ID → ITEMS.ID、ORDER_ITEM_ID → ORDER_ITEMS.ID の既存関係を利用する。 |
| Entity追加 | Review Entityを追加する。既存Entity設計に合わせて実装する。 |
| Entity関連 | User、Item、OrderItemとの関連付けを実装する。ただし不要な双方向Mappingは追加しない。 |
| Repository追加 | 商品ID検索、ユーザー検索、並び替え、平均評価取得用Repositoryメソッドを追加する。 |


| 項目 | 内容 |
|---|---|
| 投稿機能 | 注文詳細画面の商品名横に「レビューを書く」ボタンを追加する。 |
| 投稿条件 | 商品購入済みユーザーのみレビュー投稿可能にする。ORDER_ITEMSとORDERSを利用して購入確認を行う。 |
| 評価 | ★形式で1〜5まで選択可能。1未満、5以上の値は登録不可。 |
| コメント | REVIEW_COMMENTへ保存。入力なし（null）を許可する。 |
| 投稿日 | INSERT_DATEを利用し登録日時を表示する。 |


| 項目 | 内容 |
|---|---|
| 商品詳細画面 | 商品詳細画面下部へレビュー一覧を表示する。 |
| 表示内容 | 投稿者名、評価（★）、コメント、投稿日を表示する。 |
| 初期表示順 | 新しいレビュー順で表示する。 |
| 並び替え | 評価の高い順、評価の低い順へ変更可能にする。 |
| 表示件数 | 最新5件を表示する。「全件表示」ボタンで全レビュー表示する。 |


| 項目 | 内容 |
|---|---|
| トップ画面 | 商品一覧画面に平均評価とレビュー件数を表示する。 |
| 評価計算 | 商品ごとのレビュー平均値を計算して表示する。 |
| 表示形式 | 平均評価を★形式で表示する。 |


| 項目 | 内容 |
|---|---|
| 削除仕様 | 自分が投稿したレビュー、またはADMIN権限ユーザーのみ削除可能。 |
| 編集仕様 | レビュー編集機能は実装しない。 |
| 権限確認 | 既存AUTHORITYまたはSession情報を利用して判定する。 |


| 項目 | 内容 |
|---|---|
| 受け入れ条件 (Acceptance Criteria) | 注文詳細画面からレビュー入力画面へ遷移できること。 |
| 受け入れ条件 | 未購入の商品にはレビュー投稿できないこと。 |
| 受け入れ条件 | 投稿したレビューが商品詳細画面へ表示されること。 |
| 受け入れ条件 | 投稿者名、評価、コメント、投稿日が表示されること。 |
| 受け入れ条件 | 評価1〜5以外の値が登録されないこと。 |
| 受け入れ条件 | コメント未入力でも登録できること。 |
| 受け入れ条件 | 新しい順で初期表示されること。 |
| 受け入れ条件 | 高評価順、低評価順へ並び替えできること。 |
| 受け入れ条件 | 商品一覧で平均評価とレビュー件数が表示されること。 |
| 受け入れ条件 | 投稿者本人またはADMINのみ削除できること。 |
