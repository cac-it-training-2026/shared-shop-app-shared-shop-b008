# アカウントロック機能の実装

## 概要

不正ログイン（ブルートフォースアタック）対策として、ログイン失敗回数を管理し、5回連続で認証に失敗したアカウントを30分間ロックする機能を実装する。

管理者・一般会員の両方を対象とする。

---

## 背景

現行システムではログイン試行回数に制限がなく、パスワード総当たり攻撃への対策が行われていない。

クライアント要望に基づき、アカウントロック機能を追加しセキュリティを向上させる。

---

## 技術的制約・前提条件 (Constraints & Context)

- Spring Securityなどpom.xml変更を伴うライブラリは使用しないこと
- サービスレイヤは作成しないこと
- ControllerからRepository（Spring Data JPA）を直接呼び出す既存構成を維持すること
- テンプレートエンジンはThymeleafを利用すること
- DBアクセスはSpring Data JPAを利用すること
- 認証情報はHttpSessionで管理すること
- Entity変更を許可する
- DB変更を許可する
- DB変更時はDDL(SQL)を成果物へ含めること
- 既存プロジェクトのコーディング規約およびGoogle Java Styleに準拠すること
- ブランチ名は `feat#<Issue番号>` とすること

---

## 実装禁止事項

- Spring Securityを導入しないこと
- pom.xmlを変更しないこと
- 新たな認証フレームワークを追加しないこと
- サービスレイヤを新規作成しないこと
- 認証処理をFilterへ移動しないこと
- 既存のLoginControllerを中心に改修すること

---

## 要件

### ログイン失敗時

- loginIdが存在する場合のみ失敗回数を記録する
- loginIdが存在しない場合は失敗回数を記録しない
- ログイン失敗時は `login_failure_count` を1加算する
- エラーメッセージは以下に統一する

```text
ログインIDまたはパスワードが違います
```

### アカウントロック

- 5回連続でログインに失敗した場合、アカウントをロックする
- ロック時に `lock_until` へ「現在時刻 + 30分」を保存する

### ロック中

- ロック中は正しいパスワードを入力してもログインできないこと
- 以下のメッセージを表示すること

```text
ログインに5回失敗したため、
30分間アカウントがロックされています。
```

- ロック中のログイン試行では `login_failure_count` を更新しないこと

### ロック解除

- `lock_until` を過ぎた場合は自動解除する
- バッチ処理は利用しない
- ログイン処理実行時に解除判定を行う
- ロック解除時は失敗回数を0へリセットする

### ログイン成功時

- `login_failure_count` を0へリセット
- `lock_until` をNULLへ更新

## DB変更

### usersテーブル

| カラム名 | 型 | 内容 |
|----------|----------|----------|
| login_failure_count | INT | ログイン失敗回数 |
| lock_until | DATETIME | ロック解除日時 |

### DDL作成

以下を成果物として作成すること

- ALTER TABLE文
- ロールバック用SQL
- テストデータ投入SQL（必要に応じて）

※ テーブル名・データ型は既存DB定義に合わせて調整すること

## 受入条件 (Acceptance Criteria)

- [ ] loginIdが存在しない場合は失敗回数を記録しない
- [ ] loginIdが存在しない場合でもエラーメッセージは「ログインIDまたはパスワードが違います」と表示される
- [ ] 1～4回目の失敗で失敗回数が加算される
- [ ] 4回失敗後にログイン成功すると失敗回数が0になる
- [ ] 5回目の失敗でアカウントがロックされる
- [ ] ロック時にlock_untilへ30分後の日時が保存される
- [ ] ロック中は正しいパスワードでもログインできない
- [ ] ロック中は指定メッセージが表示される
- [ ] ロック中のログイン試行ではlogin_failure_countが更新されない
- [ ] 30分経過後にログイン可能となる
- [ ] ロック解除時に失敗回数が0へリセットされる
- [ ] ADMINユーザーでも同様に動作する
- [ ] USERユーザーでも同様に動作する
